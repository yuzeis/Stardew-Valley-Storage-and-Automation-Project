Stardew Valley Storage and Automation Project (SVSAP) 1.2.0-alpha.2 使用教程
源码：https://github.com/yuzeis/Stardew-Valley-Storage-and-Automation-Project

一、安装与语言

1. 安装 Stardew Valley 1.6.15 与 SMAPI 4.5.2 或更新版本，并使用 SMAPI 启动游戏。
2. 把发布包里的 SVSAP 文件夹放入 Stardew Valley/Mods/。
3. 默认语言是中文。要切换英文，打开 SVSAP/config.json：
   "Language": "en"
   改回中文：
   "Language": "zh"
4. 如果安装了 Generic Mod Config Menu，也可以在模组配置页切换语言。

二、最快上手

1. 制作网络核心、网络终端、链接工具、存储接口或存储驱动器。
2. 放置网络核心。
3. 手持链接工具右键网络核心，选择这个网络。
4. 手持同一把链接工具，右键网络终端、存储接口、存储驱动器、导入器、导出器等设备，把它们加入网络。
5. 存储接口旁边放箱子，或在存储驱动器里插入物品存储元件。
6. 右键网络终端，就可以搜索、取出、存入网络里的物品。

三、基础物品

基础电路、进阶电路、高级电路：
用于制作 SVSAP 机器和更高等级组件。

网络线缆：
在要求线缆连接时，用于连接网络设备。默认配置允许农场内简易无线。

链接工具：
先右键网络核心选择网络，再右键其他设备加入该网络。

空白样板：
在样板终端中编码成合成样板或处理样板。

物品存储元件：
插入存储驱动器，提供数字化库存容量。容量等级从 1K 到 4096K。

过滤卡、容量升级卡、速度升级卡、品质控制卡：
用于配置导入器和导出器。过滤卡切换白名单/黑名单/清空过滤；速度卡提高搬运频率；容量卡提高每次搬运数量；品质控制卡切换材料品质策略。

四、机器作用

网络核心：
一个网络的主控设备。链接工具必须先选中网络核心，其他设备才能加入这个网络。

网络终端：
打开全网物品界面。可以搜索、按分类筛选、按品质筛选、取出、存入、锁定物品。

存储接口：
把相邻箱子接入网络。适合前期直接用箱子作为网络库存。

存储驱动器：
插入物品存储元件后提供数字化库存。右键可插入或弹出存储元件。

导入器：
从相邻箱子或机器把物品搬入网络。可以设置过滤物品和来源保留数量。

导出器：
从网络把指定物品输出到相邻箱子或机器。可以设置过滤物品和目标保有数量。

合成终端：
从网络库存直接执行普通合成。材料会从网络扣除，成品会存回网络。

样板终端：
把配方写入空白样板。可写入普通合成样板，也可写入机器处理样板。

样板供应器：
存放样板，供自动合成和处理流水线使用。

分子装配器：
执行普通合成样板。自动合成需要它来生产普通合成结果。

合成 CPU 核心：
自动合成请求的主控。至少需要一个 CPU 核心才能排队自动合成任务。

合成矩阵：
提高自动合成任务容量。等级越高，可承载的任务越大。

协处理器：
提高单个合成流程内的并行发派能力，让复杂任务更容易同时推进多个步骤。

机器接口：
处理样板使用的机器桥接设备。它会把材料投入相邻机器，并在机器完成后收取产物。

合成监视器：
查看 CPU 作业、取消作业、管理处理流水线和陈酿流水线。

五、多人联机

所有玩家必须安装同一版本 SVSAP。使用 SVSAPME 时，主机与客机也必须安装同一版本 SVSAPME。主机保存网络数据，客机操作网络设备时会向主机发送请求。建议联机前确认每个人的 config.json 中 Language 可按个人喜好设置，但模组版本必须一致。

六、常用提示

如果终端里看不到物品：
检查设备是否已用链接工具加入同一个网络；检查箱子是否相邻；检查存储驱动器是否插入了存储元件。

如果导入器或导出器不搬运：
检查过滤设置、相邻目标、来源保留数量、目标保有数量，以及网络是否有可用空间。

如果自动合成无法排队：
检查是否有合成 CPU 核心、合成矩阵、分子装配器、样板供应器和对应样板；再检查网络库存材料是否足够。
